# Portfolio Summary

## Project

A self-hosted Proxmox remote development platform designed for:
- repeatable provisioning
- automated configuration
- GPU and CPU development workloads
- shared storage
- remote access
- disaster recovery and rebuild readiness

## Why this project matters

This is more than a homelab. It demonstrates:
- infrastructure design
- systems thinking
- automation discipline
- separation of code, secrets, and state
- recovery planning
- platform documentation for future maintainers

## Technical themes
- Proxmox virtualization
- OpenTofu infrastructure as code
- Ansible configuration management
- SSH-based operations
- Tailscale remote connectivity
- NFS shared storage
- GPU passthrough for specialized workloads

## Engineering outcomes
- reduced rebuild uncertainty
- reduced manual configuration drift
- improved portability of the environment
- a clearer path from live system to reproducible platform

## What makes it portfolio-ready
- architecture documentation
- explicit recovery strategy
- documented tradeoffs
- clean repository hygiene
- real-world constraints instead of toy examples
